## 🔄 3. Изменения в промптах и правилах

### 1. Изменение флоу диалога (`conversation_flow.md`) — ⚠️ ВАЖНОЕ ЗАМЕЧАНИЕ
* **В первой базе (`test_cayu`)** настроен рабочий флоу `Hotel Individual Booking`, который содержит **очень детальный глобальный промпт** (`global_prompt`). В нем прописаны жесткие ограничения системы: приоритеты языков общения (русский, кыргызский, английский), запрещенные фразы для ИИ (не говорить «бронирую», не давать номер брони), правила работы с ценами и форматирование стиля ответов.
* **Во второй базе (`test_cayu_new`)** этот глобальный промпт **полностью пустой** (`Test Reset Flow` / `temp`). Хотя многие правила были перенесены в промпты отдельных агентов, общие системные ограничения верхнего уровня из глобального флоу здесь отсутствуют. Если возникнут проблемы с поведением ассистента, эти правила стоит взять из первой базы.

```diff
-## Flow: Hotel Individual Booking
-*Guides a lead from first contact through room selection and booking confirmation.*
+## Flow: Test Reset Flow
+*temp*
```

### 2. Детализация промптов агентов (`prompts\*.md`)
Промпты агентов `booking`, `consultant` и `cs` во второй базе были переведены с английского языка на русский и получили детальную структуру с пошаговыми алгоритмами:

* **Booking Agent (`booking_agent.md`):**
  Добавлены разделы: «ПЕРЕД КАЖДЫМ ОТВЕТОМ — ОБЯЗАТЕЛЬНО», «ЯЗЫК — АБСОЛЮТНЫЙ ПРИОРИТЕТ» и четко расписанные сценарии для шагов.
* **Consultant Agent (`consultant_agent.md`):**
  Добавлен пошаговый алгоритм квалификации гостей («Шаг 1 — Задай ОДИН квалификационный вопрос», «Шаг 2 — Сделай рекомендацию»).
* **Customer Service Agent (`cs_agent.md`):**
  Добавлен список вопросов, на которые отвечает агент (бассейн, питомцы, парковка), и критическое правило для перевода на бронирование.
* **Intent Router (`router_agent.md`):**
  Добавлено правило классификации запроса фотографий номера как запрос бронирования:
  ```diff
  +— Запрос фото номера = booking, не faq
  +— «do you have pictures», «show me photos», «покажи фото»,
  +  «can I see the room» → всегда booking
  ```
